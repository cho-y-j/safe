## 3. 핵심 서비스 로직 (SensorService.kt)

### 3.1 워치 앱 흐름

```
[1] SetupActivity
    → 작업자 ID + 서버 주소 + 작업 유형 선택
    → 작업 유형별 경보 범위 설정 (연구 기반)
      사무직: ±40, 경량: ±55, 중량: ±65, 야외: ±80

[2] CalibrationActivity (10분)
    → "학습 시작" 버튼 눌러야 시작
    → WakeLock으로 화면 꺼져도 센서 유지
    → 안정/활동 심박 분류 수집
    → 완료: "안정 78bpm, 활동 87bpm, 경보 133↑/48↓"

[3] MainActivity (3화면)
    → 대기: [▶ 작업 시작] + [⚙ 설정]
    → 작업중: 심박 + AI 상태 + [🔋 휴식] + [⏹ 종료]
    → 휴식중: 타이머 + 목표시간 + [▶ 작업 복귀]
```

### 3.2 SensorService 6단계 안전 흐름

```
1. 개인 베이스라인 학습 (EMA, 안정/활동 분리)
2. 이상 감지 (연구 기반 ± 범위)
   상한: 안정평균 + 직업범위 초과
   하한: 안정평균 - 30 미만
   절대: 180↑ 또는 40↓
   급락: 20bpm 이상 급강하
3. 본인 알림 + 확인 버튼 (5초 대기)
   "괜찮아요" → 정상으로 5배 가중 학습 (개인화)
4. 잠 vs 응급 AI 구분
   움직임 없음 + 바이탈 정상 = 수면
   움직임 없음 + 바이탈 이상 = 응급
5. P2P BLE 경보 + 서버 동시 전송 (이중 경로)
   A(d) = Amax × (1 - d/Dmax) 거리비례 진동
6. 90초 무응답 → 119 자동 연동 (TODO)
```

### 3.3 5단계 적응형 모니터링 (연구 기반)

```
1. IDLE_REST:    60초 간격 (배터리 48시간)
2. ACTIVE:       30초 간격 (활동 감지 시)
3. CHANGE_DETECT: 15초 간격 (심박 ±15 급변)
4. ALERT_NEAR:   5초 간격 (경보 범위 80% 도달)
5. ALERT_OVER:   1초 간격 (경보 초과 → 즉시 알림)
```

### 3.4 추가 감지 기능

```
낙상 감지: 가속도 자유낙하(<3) + 충격(>25) → 5초 확인 → 응급
워치 탈착: 정상→심박0 = 벗음(조용히), 이상→심박0 = 먼저 물어봄
법정 휴식: 4시간 근무 → 30분 휴식 알림 (근로기준법 54조)
```

## 4. 워치↔폰 통신

### 4.1 Wearable Data Layer API

```
워치 → Wearable.getMessageClient().sendMessage() → 폰
(기존 블루투스 연결 활용, 별도 BLE 불필요)

경로:
  /safepulse/status  → 센서 데이터 (60초마다)
  /safepulse/alert   → 긴급 경보
  /safepulse/ack     → 확인 응답

중요: 워치와 폰 앱의 패키지명이 동일해야 함!
  워치: com.dainon.safepulse
  폰:   com.dainon.safepulse (서브패키지: .phone)
```

### 4.2 P2P BLE (워치↔워치)

```
BLE Advertise/Scan으로 워치끼리 직접 통신
Manufacturer ID: 0xDADA (다인온)
광고 데이터: "SPW-001N" (Normal) / "SPW-001E" (Emergency)
RSSI → 거리 추정 → 진동 강도 비례
서버 없이도 동작 (특허 핵심)
```
