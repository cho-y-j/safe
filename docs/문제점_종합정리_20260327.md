# SafePulse 문제점 종합 정리 + 해결 방안 (2026-03-27)

---

## A. 발신자(사고자) 워치 문제

### A-1. AckAlertActivity가 사라짐 — 괜찮아요/오작동 누를 수 없음
**현상:** AckAlertActivity가 뜨지만 화면 터치하면 MainActivity로 전환되어 사라짐
**원인:** `taskAffinity=""`로 별도 태스크 → 다른 태스크 전환 시 뒤로 밀림
**해결:** taskAffinity 제거 + `FLAG_ACTIVITY_CLEAR_TOP`으로 MainActivity 위에 올리기. AckAlertActivity가 항상 최상단.

### A-2. 발신자 진동/비프가 매 1초 리셋됨
**현상:** EMERGENCY 루프가 1초마다 emergencyVibrate() 재호출 → 진동 끊겼다 다시 시작
**원인:** evaluateState() EMERGENCY case에서 매번 호출
**해결:** `isEmergencyAlarmActive` 플래그 추가. 이미 울리고 있으면 재호출 안 함. ACK/Dismiss 시에만 정지.

### A-3. 폰에서 "문제 없음" 눌러도 워치 진동 안 멈춤
**현상:** S22에서 종료 눌러도 Watch 4 계속 진동
**원인:** 폰→워치 ACK가 Wearable Data Layer로 전달되는데, SensorService가 EMERGENCY 루프에서 즉시 재에스컬레이션
**해결:** 폰 ACK 수신 시 `ackCooldownUntil` 설정 + 비프/진동 즉시 정지. 폰 앱에도 "오작동 멈추기" 버튼 추가.

### A-4. 낙상 감지 임계값 너무 높음
**현상:** 워치를 집어던져야 할 정도로 감지 안 됨
**원인:** `FREE_FALL_THRESHOLD = 3.0f`, `IMPACT_THRESHOLD = 25.0f` → 너무 높음
**해결:** 테스트용 임계값 대폭 낮춤
- FREE_FALL_THRESHOLD: 3.0 → 5.0 (자유낙하 판정 완화)
- IMPACT_THRESHOLD: 25.0 → 15.0 (충격 판정 완화)

## B. 수신자(주변인) 워치 문제

### B-1. 거리 실시간 업데이트 안 됨
**현상:** 5m까지 가도 화면에 3m 고정. 진동 간격도 안 바뀜
**원인:** 30초 중복 방지가 진동+거리 업데이트 모두 차단
**해결:** 분리 — 진동 시작은 첫 1회만, **거리/zone은 매 BLE 수신마다 갱신** + 진동 패턴도 zone 변경 시 즉시 교체

### B-2. P2pAlertActivity 거리 갱신 안 됨
**현상:** Activity 화면에 처음 거리 고정
**원인:** onNewIntent() 없음 → 새 거리 수신 못함
**해결:** P2pAlertActivity에 onNewIntent() 추가 → 거리/zone 실시간 업데이트

### B-3. TX_POWER 캘리브레이션
**현상:** 실제 5m인데 3m로 표시
**원인:** TX_POWER=-59는 일반값. Galaxy Watch 4/5 실제값과 다를 수 있음
**해결:** TX_POWER를 -65로 조정 (워치 BLE 출력이 폰보다 약함)

## C. 폰 앱 문제

### C-1. S22에서 Watch 5 긴급 시 지도 안 뜸
**현상:** Watch 5 낙상 → S22에서 아무 반응 없음
**원인:** S22의 BridgeService BLE 스캔 로그 자체가 없음 → 서비스 미실행 또는 BLE 감지 실패
**해결:** BridgeService 시작 로그 강화 + BLE 스캔 상태 확인. 앱 시작 시 서비스 확실히 실행.

### C-2. 폰에 "오작동 멈추기" 버튼 없음
**현상:** 긴급 카드에 "문제 없음" + "119"만 있음
**해결:** 발신자 폰 긴급 카드에 "오작동 멈추기" 버튼 추가 → 워치에 DISMISS 전달

### C-3. 폰 백그라운드 자동 실행
**현상:** 앱 꺼놓으면 BLE 수신 불가
**해결:** START_STICKY (이미 추가) + 폰 재부팅 시 자동 시작 (BootReceiver)

## D. 대시보드 문제

### D-1. 긴급 팝업이 안 뜨거나 늦게 뜸
**현상:** 워치 EMERGENCY 시 대시보드에 팝업 안 뜸
**원인:** realWatchData에 status 전달은 수정했지만 테스트 미완
**해결:** emergency Socket.io 이벤트 직접 수신 (이미 코드 추가) + 서버 재시작 후 테스트

## E. 거리 계산 + 진동 간격 시스템

### E-1. 현재 구조 문제
```
BLE 수신 → 거리 계산 → 30초 중복 방지 → 진동+화면 한 번만
```
30초간 거리가 변해도 반영 안 됨.

### E-2. 변경 후 구조
```
BLE 수신 → 거리 계산 → 매번 zone 갱신
  → zone 변경 시: 진동 패턴 교체 + 화면 거리 업데이트
  → zone 동일 시: 화면 거리만 업데이트 (진동 유지)
  → P2pAlertActivity/fullScreenIntent: 첫 1회만
```
