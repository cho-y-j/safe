## 1. 아이디어 내용 요약

인천공항에는 320여 개 상주기업, 약 **7만 명의 종사자**가 활주로, 계류장, 화물터미널 등에서 근무합니다(출처: 인천공항 상주기업 현황). 폭염·한파·미세먼지 등 환경 위험과 과로·낙상 등 산업재해 위험에 상시 노출되어 있으나, 현재의 안전 관리는 **사고 발생 후 대응**에 머물러 있어 골든타임을 놓치는 경우가 빈번합니다.

**SafePulse**는 스마트워치(Galaxy Watch)의 심박·체온·SpO₂·가속도 센서와 **AI 개인 맞춤 학습**을 결합하여, 작업자의 이상 징후를 **사전에 단계적으로 감지**하고, **서버 없이도 BLE P2P(Bluetooth Low Energy)로 30m 반경 동료에게 즉시 경보**를 전파하는 시스템입니다. 관제 대시보드에서는 공공데이터(기상·대기질·항공편)와 생체신호를 융합한 AI 위험도 분석으로 선제적 안전 관리가 가능합니다.

## 2. 아이디어 제안 내용

### 1) 제안 배경 및 목적

#### 현장의 문제

인천공항은 연간 **7,000만 명 이상**의 여객이 이용하는 세계 5위 공항으로, 24시간 운영되는 활주로·계류장·화물터미널에서 작업자들이 극한 환경에 노출됩니다.

| 위험 요소 | 현황 |
|----------|------|
| **폭염** | 인천공항은 해안 입지로 여름 체감온도 35°C 이상 빈번. 정부 기준 체감온도 33°C 이상 시 2시간마다 20분 이상 의무 휴식 필요 |
| **한파·미세먼지** | 겨울철 해풍으로 인한 체감온도 급락, 봄철 미세먼지(PM2.5) 고농도 빈번 |
| **과로** | 항공편 집중 시간대(오전 8~10시, 오후 2~4시) 수하물·화물 작업량 급증 |
| **낙상·충돌** | 활주로·계류장의 넓은 실외 공간에서 쓰러진 작업자를 발견하기까지 수분~수십분 소요 |

#### 기존 한계

1. **사후 대응**: 사고 발생 후에야 인지. 심정지 골든타임(4분) 내 처치율 극히 낮음
2. **개인차 무시**: 심박 120bpm이 사무직 관제사에겐 위험이지만 화물 하역 작업자에겐 정상. 일률적 기준으로는 오경보 과다 또는 실제 위험 누락
3. **통신 사각지대**: 활주로·격납고 등 Wi-Fi 불안정 구역에서 서버 기반 모니터링 불가능
4. **단일 시점 감시**: 문제 발생 시에만 반응. 주의→경고→위험으로의 단계적 예측 부재

#### 법적·사회적 배경

**중대재해처벌법**(2022년 시행)에 따라 사업장 내 사망사고 발생 시 경영책임자에게 **1년 이상 징역 또는 10억원 이하 벌금**이 부과됩니다. 인천공항공사를 비롯한 320여 상주기업 경영진에게 작업자 안전은 **법적 의무**이자 경영 리스크입니다. SafePulse는 이 법적 의무를 기술로 이행하는 솔루션입니다.

또한 작업자 안전사고는 **항공편 지연·취소**로 직결되어 수천 명의 여객에게 피해를 줍니다. 활주로 작업자 사고 → 활주로 폐쇄 → 이착륙 중단 → 대규모 지연. **작업자 안전이 곧 국민(여객) 안전**입니다.

#### 목적 — 5대 핵심 가치

| # | 핵심 가치 | 설명 |
|---|---------|------|
| 1 | **AI 개인화 학습** | 작업자별 베이스라인 학습 + 지속 업데이트 → 오경보 최소화, 정밀 감지 |
| 2 | **P2P 즉시 경보** | 서버 없이 BLE로 가장 가까운 동료에게 즉시 알림 + 폰 지도 + 제세동기 위치 |
| 3 | **단계적 감시 강화** | 정상→주의→경고→위험 단계별 점검 주기 자동 조절 + 문제 예측 |
| 4 | **공공데이터 융합** | 기상·대기질·항공편으로 업무량 예측 + 환경 위험 사전 경보 + 실내 위치 파악 |
| 5 | **배터리 최적화** | 5단계 적응형 모니터링으로 8시간 근무 중 방전 방지 |

### 2) 상세 설명

#### SafePulse만의 차별점 — 기존 안전 시스템과의 비교

| 항목 | 기존 안전 시스템 | **SafePulse** |
|------|-------------|-------------|
| **경보 방식** | 서버로 알림 → 관제실 확인 → 현장 출동 (**수분 소요**) | **P2P BLE로 가장 가까운 동료에게 즉시 전파 (1초)** |
| **통신 두절 시** | **작동 불가** (서버 의존) | **서버 없이 단독 작동** (BLE P2P 오프라인 안전망) |
| **개인차 대응** | 일률적 기준 → 오경보 과다 또는 위험 누락 | **AI 개인화 학습 + 지속 업데이트 + 피드백 반영** |
| **거리 측정** | 없음 (위치만 지도 표시) | **BLE 거리비례 진동 → 전용 디바이스 UWB 10cm 정밀도** |
| **누가 구조하나** | 관제실이 지정 → 출동 (느림) | **거리비례 진동 → 가장 가까운 사람이 자동으로 찾아감** |
| **감시 방식** | 문제 발생 시에만 반응 | **5단계 적응형: 정상→주의→경고→위험 점진적 강화** |
| **전력** | 범용 워치 (1일) | **저전력 설계: 전용 디바이스 7일+, 적응형 모니터링** |
| **AED 연동** | 없음 | **가장 가까운 제세동기 위치 자동 표시 (78개소)** |

> 긴급 상황에서 서버를 거쳐 관제실이 출동 지시하는 기존 방식은 **수분이 소요**됩니다. SafePulse는 통신이 끊어져도 **가장 가까운 동료가 1초 만에 인지하고 즉시 달려갑니다.** 이것이 골든타임(4분)을 살리는 핵심 차이입니다.

#### 시스템 구조

```
[작업자 워치] ──BLE P2P──→ [주변 동료 워치] (서버 없이 즉시 작동)
      │                           │
      │←──Bluetooth──→ [작업자 폰]  [동료 폰] ←─Bluetooth─→│
      │                           │              │
      └───WiFi/LTE───→ [서버] ←──Socket.io──→ [관제 대시보드]
                          │                      ├ 실시간 위치/상태
                    [AI 분석 엔진]                ├ 제세동기(AED) 위치
                    + 공공데이터 융합             └ 층별 지도
```

#### 핵심 1: AI 개인화 학습 + 지속 업데이트

작업자가 워치를 착용하면 **10분간 캘리브레이션**을 수행합니다. 걷고, 앉고, 팔을 움직이는 실제 작업 동작을 하면서 **심박·체온·SpO₂·가속도계의 개인별 베이스라인**을 학습합니다.

- **학습 방법**: EMA(지수이동평균, α=0.05)로 안정/활동 시 심박을 분리 학습. 시간대별(아침/오후/저녁/야간) 패턴도 구분하여 서버에 30분마다 동기화
- **직업별 경보 범위**: NIOSH(미국 국립산업안전보건연구소)의 HRR(심박수 예비량) 기준과 PMC 건설 노동자 실측 연구를 기반으로 차등 적용

| 직업 유형 | HR 경보 상한 | SpO₂ 범위 | 체온 범위 | 연구 근거 |
|---------|-----------|----------|---------|---------|
| 사무/관제 | 안정+40bpm | -3% | ±0.5°C | NIOSH HRR 33% 기준 |
| 경량 작업 | 안정+55bpm | -4% | ±0.8°C | PMC4847018 실측 |
| 중량 작업 | 안정+65bpm | -5% | ±1.0°C | PMC4847018 과로 그룹 |
| 야외 고강도 | 안정+80bpm | -6% | ±1.5°C | 열사병 위험도 연구 |

- **"괜찮아요" 피드백 학습**: 오경보 시 사용자가 "괜찮아요"를 누르면 현재 수치가 **5배 가중**으로 베이스라인에 반영 → 다음에 같은 수치에서 경보가 울리지 않음 (개인화 강화)
- **가속도계 적응형 학습**: 10분 캘리브레이션에서 가속도 magnitude의 mean/std를 학습하여 **기기별 센서 감도 차이를 자동 보정** (Galaxy Watch 4/5 등 다양한 기기에서 동일 성능)

> **참고 연구**: Heavy Physical Work: Cardiovascular Load in Male Construction Workers (PMC4847018), Heart Rate Variability-Based Physical Fatigue Assessment (MDPI Sensors 22(9):3199, 피로 감지 정확도 93.5%), Wearable Sensors for Healthcare of Industrial Workers (MDPI Electronics 13(19):3849)

#### 핵심 2: P2P BLE 거리비례 경보 — 가장 빠르게 도움줄 수 있는 사람에게

**서버나 Wi-Fi 없이** 워치 간 직접 BLE 통신으로 **30m 반경 동료에게 1~100ms 이내** 경보를 전파합니다. 이는 통신 인프라가 불안정한 활주로·격납고에서도 작동하는 **오프라인 안전망**입니다.

**거리비례 진동**: RSSI(수신 신호 강도) → 경로손실 모델(A(d) = A_max × (1 - d/D_max))로 거리 추정 → EMA 스무딩으로 노이즈 제거 → 5구간 진동 강도 차등

| 거리 | 진동 패턴 | 의미 |
|------|---------|------|
| ~1m | 100ms on / 50ms off (빠른 연속) | 즉시 근접 — 사고자 바로 옆 |
| ~3m | 200ms on / 300ms off | 매우 가까움 — 몇 걸음 |
| ~5m | 200ms on / 1초 off | 가까움 — 소리로 방향 확인 |
| ~10m | 200ms on / 2초 off | 중간 — 주변 탐색 |
| 10m+ | 200ms on / 4초 off | 멀리 — 이동 필요 |

**3자 관점 완전 연동**:
- **사고자 워치**: 최대 볼륨 비프음 연속 재생 → 소리 방향으로 찾아감
- **동료 워치**: 거리비례 진동 + "응답함"/"도움불가" 버튼
- **동료 폰**: 사고자 위치 지도 즉시 확인 + Google Maps 길안내 + **가장 가까운 제세동기(AED) 위치 표시** (인천공항 78개소 데이터)
- **관제 대시보드**: 사고자 위치 자동 줌 + 층별 지도 + P2P 응답 상태 실시간 확인 + AED 위치

**3중 통신 보장**: P2P BLE(1~100ms, 서버 불필요) + 워치→폰→서버(100~500ms, LTE) + 워치→서버(Wi-Fi) — 하나라도 성공하면 전달

#### 핵심 3: 단계적 감시 강화 — 5단계 적응형 모니터링

문제 발생 시에만 감시하는 것이 아니라, **정상→주의→경고→위험으로 단계적으로 점검 주기가 빨라지고** 문제 발생을 예측합니다.

| 단계 | 상태 | 센서 체크 | 서버 전송 | 조건 |
|------|------|---------|---------|------|
| 1 안정 | 정상 | 5초 | 5분 | 활동 없음, HR 정상 |
| 2 활동 | 정상 (활동 중) | 5초 | 5분 | 가속도 감지, HR 약간 상승 |
| 3 변화 감지 | **주의** | **3초** | **30초** | HR 급변(±15bpm) 감지 |
| 4 경보 근접 | **경고** | **2초** | **10초** | HR이 경보 범위 80% 도달 |
| 5 경보 초과 | **위험** | **1초** | **즉시** | HR 경보 초과 or 낙상 감지 |

- **주의 단계**: 본인에게 "컨디션을 확인해주세요" 알림. 관제 대시보드 색상 변경(파란색)
- **경고 단계**: "휴식을 권고합니다" + 관제 대시보드 주황색 + 관리자 알림
- **위험 단계**: "넘어지셨나요?" 전체화면 확인 → 5초 무응답 시 **P2P BLE 경보 자동 발동** + 관제 팝업

#### 핵심 4: 공공데이터 활용 — 업무 예측 + 환경 경보 + 위치 파악

| 공공데이터 | 출처 | 활용 |
|----------|------|------|
| 실시간 대기질 (PM2.5, O₃, CO, NO₂, SO₂) | data.go.kr 에어코리아 | 대기질 악화(PM2.5≥35) → 실외 작업자 마스크 착용 + 호흡기 질환자 실내 배치 |
| 초단기 기상실황 (기온, 체감온도, 풍속) | data.go.kr 기상청 | 폭염(33°C↑) → 작업 로테이션 30분 단축, 한파 → 방한 장비 경보 |
| 실시간 항공편 운항 현황 | openapi.airport.co.kr | 항공편 집중 시간대 → 과로 예측 → 사전 인력 재배치 |
| 시간대별 여객 예보 | data.go.kr 인천공항 | 혼잡 구역 예측 → 작업자 피로도 사전 관리 |
| 월별 항공 통계 | data.go.kr 항공통계 | 장기 트렌드 → 계절별 인력 배치 최적화 |

**AI 5개 분석 모듈**:
1. **다중 인자 위험도 융합** (기상 40점 + 대기질 35점 + 항공편 25점 + 생체 30점 → 0~100점)
2. **집단 바이오시그널 환경재해 감지** (같은 구역 3명+ 동시 이상 → 가스누출/산소부족 자동 판별)
3. **구역별 혼잡도 4시간 예측** (항공편 + 여객 + 시간대 가중)
4. **개인 피로도 예측** (HRV 추세 + 근무시간 + 환경 → 피로도 0~100점, 연구 기반 정확도 93.5%)
5. **LLM 자연어 인사이트** (DeepSeek → "오전 10시 화물터미널 인력 부족 예상, 2명 추가 배치 권고" 등 한국어 권고사항 자동 생성)

**실내 위치 파악**: BLE RSSI 기반 구역 수준 위치 추정 + GPS로 실외 위치 추적. 관제 대시보드에서 **터미널별·층별 지도**에 작업자 위치 실시간 표시. BLE 실내 측위는 현재 1~3m 정확도 달성 가능(출처: ScienceDirect, BLE indoor positioning, 2025)

#### 핵심 5: 배터리 최적화 — 8시간 근무 중 방전 방지

워치 배터리 소모의 **50%는 Wi-Fi/LTE 통신**입니다. SafePulse는 통신 빈도만 상황에 따라 조절하고, 센서는 항상 빠르게 체크합니다.

| 항목 | 평상시 | 이벤트 시 | 배터리 비중 |
|------|--------|---------|----------|
| 센서 체크 | 5초 | 1초 | 15% (항상 저전력) |
| **서버 전송** | **5분에 1회** | **즉시 (1초)** | **50% → 최소화** |
| 폰 전송 | BT, 매 루프 | 즉시 | 10% (BLE 기반) |
| BLE P2P 광고 | LOW_POWER | HIGH (최대출력) | 15% |
| 화면/진동/소리 | 꺼짐 | 최대 | 이벤트 시만 |

**예상 배터리 수명**: 평상시 **24시간+** (8시간 근무 충분), 이벤트 시 4시간+ (긴급 상황 대응 충분)

**워치 미착용 시 자동 절전**: HR=0 감지 → 모든 기능 최저 전력 → 10분 후 자동 작업 종료 → 서비스 정지 → 배터리 소모 0
