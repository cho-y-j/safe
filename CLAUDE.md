# SafePulse 프로젝트 지침

## 개요
인천공항 작업자 안전 관리 시스템. Galaxy Watch로 바이탈 모니터링 → 이상 감지 → P2P BLE 경보 → 관제 대시보드.
공모전 마감: 2026-04-03

## 3자 관점 (절대 혼동 금지)

### 발신자 (사고자) — Watch + Phone
- 본인에게 이상 감지됨
- **워치 화면**: "괜찮아요" (학습+해제) / "즉시 호출" (바로 P2P+서버 경보)
- **폰**: 긴급 상태 표시 + "문제 없음" 버튼. 지도 필요 없음 (본인이 쓰러졌는데 지도를 왜 봄)
- 무응답 5분 → 자동 EMERGENCY

### 수신자 (주변 동료) — Watch + Phone
- 다른 작업자의 사고를 BLE로 수신
- **워치**: 화면 자동 켜짐 + 풀스크린 "누구 + 거리" + 진동+비프 + "응답함"/"도움불가"
- **폰**: 사고자 위치 지도 자동 표시 + 거리 + "길 안내"
- 이 사람이 사고자를 찾아가야 하므로 **지도는 수신자에게**

### 관제자 (대시보드)
- 모든 작업자 모니터링
- 긴급 발생 → 팝업 + "지도에서 보기" → 사고자 위치 자동 줌
- P2P 응답 상태 실시간 확인

## 절대 금지 사항
1. "SafePulse" 텍스트를 워치 화면/알림에 절대 표시하지 않음
2. 의미 불명확한 버튼 금지 (예: "멈춤"). 항상 명확한 한국어 사용
3. 발신자(사고자)에게 지도 보여주기 금지
4. 수정 전 반드시 "이 화면은 누구 입장인가?" 확인
5. 계획 중에 코드 수정 금지. 반드시 허락 후 진행
6. 임기응변 금지. 항상 플랜 먼저.

## 기술 스택
- 워치: Kotlin, Wear OS, Samsung Health Services API
- 폰: Kotlin, Android, Wearable Data Layer API
- 서버: Node.js, Express, Socket.io, Sequelize
- AI: Python, FastAPI, DeepSeek LLM
- 대시보드: React, MUI, Leaflet, Socket.io
- 통신: P2P BLE (0xDADA), Wearable Data Layer, FCM

## 핵심 로직
- 개인 베이스라인 학습 (HR, SpO₂, 체온) → EMA 업데이트
- "괜찮아요" → 현재값 5x 가중 학습 → 범위 확장 → 같은 값 재경보 안 남
- 직업별 경보 범위 (사무/경량/중량/야외)
- 5단계 적응형 모니터링 (배터리 최적화)
- 잠 vs 응급 구분
- 거리비례 P2P BLE 진동 (특허 핵심)
